Introduction

🤔 What is a Taxonomy?

A taxonomy is a way to group and classify content based on shared attributes. They give your content context and make it filterable and navigable.

🏷️
Tags

On blog posts, articles, or any content.

📂
Categories

Group products, posts, or resources.

🎬
Genres

Classify movies, music, or media types.

Simple idea

Taxonomy = a system to organize content by labels

In Practice

Relationships

🔗 How Taxonomies Connect

Taxonomies don't work alone — they connect to collections. Once attached, they power filtering and listing across your site.

relationship
Collection (Blog)
 ├── Post 1 → tags: React
 └── Post 2 → tags: CSS
Filter posts by their tags
List all posts under a specific tag
Configuration

⚙️ Adding Taxonomies to a Collection

Add them to your collection's YAML config file — that's all it takes:

blog.yaml
taxonomies:
  - categories
  - tags
Fields appear automatically in all entries
Routing & template variables ready to use
Important Concept

🌍 Global Nature of Terms

This is a key concept that trips up newcomers:

Terms are Global

The React tag is the same React everywhere — it doesn't matter which collection uses it.

Need Separation?

Create a new taxonomy like product_tags if you need context-specific terms that shouldn't overlap with tags.

Automatic URLs

🌐 Routing (Automatic URLs)

Statamic automatically creates URLs for your taxonomies — no manual route configuration needed.

Global Taxonomy URLs
PageURL
All tags /tags
Single tag /tags/react
Views used
resources/views/tags/index.antlers.html
resources/views/tags/show.antlers.html
Collection-Specific URLs
PageURL
Blog tags index /blog/tags
Blog tag "React" /blog/tags/react
Note

Collection-specific URLs only show entries from that collection — not globally.

URL Identifiers

🧠 Term Value vs Slug

This part confuses many developers — here's the simple version:

Term Value
"Star Wars"
Human-readable label
→
↓
Slug
"star-wars"
URL-safe version
Rule

Slug = the URL-friendly version of the term value. Statamic generates this automatically.

Auto Conversion

🔄 Normalisation

Statamic automatically converts term values to slugs when saving entries.

In your entry
tags:
  - Star Wars
  - Tatooine
  - Droids We're Not Looking For
Becomes slugs
star-wars
tatooine
droids-were-not-looking-for
⚠️ Important Rule — All treated as the same term
Star Wars≡star wars≡StAr WaRs≡star-wars
Tip

Be consistent when naming terms. Even though Statamic normalizes them, human consistency avoids confusion in your content team.

Antlers Templates

🎨 Templating (Using Taxonomies)

🏷️ Output Terms on a Page
template.antlers.html
{{ tags }}
  {{ title }} - {{ url }}
{{ /tags }}
Output
React - /tags/react
CSS - /tags/css
🔁 Listing All Terms
tags/index.antlers.html
<ul>
  {{ terms }}
    <li><a href="{{ url }}">{{ title }}</a></li>
  {{ /terms }}
</ul>

This renders a list of all available terms — great for a tag cloud or category navigation.

📄 Listing Entries Inside a Term
tags/show.antlers.html
{{ entries }}
  <li>{{ title }}</li>
{{ /entries }}
→
Navigate to /tags/react and this will automatically list all posts tagged with React.
Watch Out

⚠️ Small Gotcha

The {{ terms }} tag needs context to work correctly.

✗ Won't work
Without context
{{ terms }}
✓ Works correctly
Inside a taxonomy page
Or with a specified collection
Best Practices

💡 Pro Tips

🔍
Use Taxonomies for Filtering & Navigation

They're built for it — tag archives, category pages, and faceted filtering all become trivial.

✏️
Keep Naming Consistent

This is critical. Inconsistent naming (React vs react vs ReactJS) creates duplicate terms and messy archives.

🧹
Don't Overload with Taxonomies

Start with 1–2 taxonomies per collection. Add more only when genuinely needed — too many gets unwieldy fast.

🔀
Separate Taxonomies for Different Contexts

If product tags and blog tags shouldn't mix, create product_tags and blog_tags as separate taxonomies.
